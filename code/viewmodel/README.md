# ViewModel Layer

B 负责这里。

ViewModel 层负责游戏逻辑和数据暴露：

- 处理 `alleyfist::GameCommand`。
- 推进 Tick、移动、攻击、跳跃、连招、受击和死亡状态。
- 处理敌人 AI、Boss 行为、碰撞、伤害、精力恢复。
- 处理遭遇战锁屏、清屏解锁、GO 提示和胜负判定。
- 维护并暴露 `alleyfist::GameSnapshot`。
- 在 `SimulationTypes.h` 中维护规则参数、AI 行为、攻击类型、攻击盒和刷怪配置等内部类型。

ViewModel 不依赖 View，也不直接绘制画面。

已经实现了下面这些功能：

- 命令处理框架
  - 已经能接收 `GameCommand`，并把输入命令缓冲到队列中。
  - 这为后续做按键缓冲、连招序列、输入状态管理打好了基础。

- Tick 驱动
  - `GameViewModel` 会把 `Tick` 命令转发给 `GameSimulation`。
  - `step()` 会推进一帧逻辑，保证游戏状态能随时间更新。

- 基础移动
  - 已经实现了左右移动。
  - 还补了上下移动对应的 `laneY` 调整，属于八方向移动的雏形。

- 跳跃逻辑
  - 已经有了简单的跳跃状态和跳跃抛物线效果。
  - 角色会从地面进入跳跃状态，并在一段时间后落回地面。

- 空中攻击
  - 在跳跃状态下触发攻击后，会进入空中攻击逻辑。
  - 生成 `JumpKick` 这类攻击盒，并参与碰撞处理。

- 攻击与伤害
  - 已经实现轻攻击和重攻击。
  - 会生成 `CombatBoxViewData`，并和敌人的 `bodyBox` 做矩形碰撞检测。
  - 命中敌人后会扣血，并更新敌人状态为 `Hurt` 或 `Dead`。

- 精力与疲劳
  - 角色攻击会消耗能量。
  - 当精力耗尽时，会进入 `playerExhausted` 状态，后续攻击会被限制。

- 连招计时器
  - 现在已经有了简单的连招窗口计时逻辑。
  - 攻击命中后会推进 `comboStep`，超时后重置。

- 敌人 AI
  - 敌人会朝玩家靠近。
  - 也会在接近时尝试攻击玩家。

- 遭遇战与关卡推进
  - 已经有了遭遇战触发逻辑。
  - 角色走到触发点后会刷出小怪并将地图锁定。
  - 当敌人清空后，会解锁并显示 GO 提示。

- 失败状态
  - 玩家血量归零时会进入 `GameOver` 状态。



- 目前已经完成了“ViewModel 的核心玩法骨架和部分完整逻辑”
- 但还没有完成“更完整的连招链、Boss 战、击倒/击飞、完整敌人包抄”等更细节内容
