# AlleyFist 下一轮迭代方案

## 1. 迭代目标

建议的核心方向是：

- 让 Boss 遭遇更有戏剧性
- 让战斗更有连击感和反馈
- 让关卡推进更有节奏和变化
- 让界面和画面更像完整游戏而不是技术 demo

---

## 2. 可优先加入的玩法功能

### 2.1 Boss 遭遇与关卡节奏增强

当前规则已经有“玩家推进到后段后生成 Boss”。下一轮可以把这一步做得更完整：

- 在 Boss 生成前增加“Boss 即将出现”的提示
- 增加 Boss 出场动画、镜头推近、屏幕轻微震动
- Boss 战期间减少普通敌人数量，避免节奏被打散
- 增加 Boss 的阶段变化，例如：
  - 第一阶段：普通攻击
  - 第二阶段：冲刺或范围攻击
  - 第三阶段：更高伤害或更快移动
- 添加“遭遇战锁屏”效果，让玩家更容易感知 Boss 的出现

这会让当前规则中的“后段 Boss 战”从一个简单触发点，变成真正的高峰剧情节点。

### 2.2 更丰富的战斗系统

当前轻攻击、重攻击、跳跃和精力系统已经有基础框架，可以继续扩展：

- 增加“闪避”动作，减少被敌人连续命中的体验
- 增加“格挡”或“硬直”反馈，让战斗更有节奏
- 让轻攻击和重攻击拥有更明显的命中差异：
  - 轻攻击：快、连出
  - 重攻击：慢、硬、爆发高
- 增加连击与打击停顿（Hit Stop），提高打击感
- 增加“受击硬直”和“击退效果”，让角色动作更有反馈
- 增加技能条或特殊技：如冲刺斩、爆发冲拳等

这会让原本只靠“靠近打一下”的系统更接近真正的动作游戏。

### 2.3 敌人类型更加丰富

当前只有普通小怪和 Boss。下一轮可以增加：

- 巡逻型敌人：不主动追击，但会在区域内来回移动
- 伏击型敌人：玩家靠近后突然冲出
- 冲撞型敌人：攻击时会短时间加速
- 远程型敌人：进行投掷/弹射攻击

同时可以考虑增加“敌人波次”设计，让关卡推进更像有层次的节奏，而不是持续刷同一类怪物。

### 2.4 补给与资源系统

为了让血条和精力条更有使用价值，可以加入：

- 血包：拾取后恢复少量生命
- 精力恢复道具：短时间内提升恢复速度
- 强化道具：短时间提升攻击力或速度

这样玩家就不会只靠纯粹的“打怪”来维持资源，而是能通过关卡探索和风险选择来提升战斗体验。

## 3. 视觉与界面美化方向

### 3.1 角色与动作表现

当前角色已经有基础绘制，但可以继续加强：

- 攻击时增加拳风/手臂延展效果
- 受伤时增加抖动、血迹、受击闪白
- 跳跃时增加重心变化，落地时有轻微震动
- 死亡时增加倒地、翻滚、碎裂或消散效果
- Boss 角色增加更明显的体型和姿态差异

### 3.2 背景与场景氛围

现有街景已经有基础视觉，但可以继续做得更有“巷战”气质：

- 街道增加路灯、招牌、垃圾桶、广告牌
- 背景加入雨滴、夜色、霓虹、烟雾、火光效果
- 场景随进度有变化：前段普通街区，中段混乱巷战，后段 Boss 现场
- 关卡尾部加入更强烈的空间压迫感，例如天桥、废弃建筑、火焰灯光

### 3.3 HUD 与 UI 设计

可以把 HUD 做得更有游戏感：

- 血条和精力条增加渐变、边框、光效
- Boss 血条改成更醒目的横幅式 UI
- 进度条增加“Boss 到来”标记点
- 结算界面增加击杀数、存活时间、连击数、评价等级
- 标题页增加“开始游戏 / 设置 / 退出”风格化按钮

---

## 4. 音效与反馈建议

虽然当前项目主要以视觉为主，但音效会极大提升沉浸感：

- 攻击命中声、重击声、脚步声、跳跃声
- Boss 出场音效、怒吼、受伤音效
- 胜负结算音效
- 轻微的环境音，比如远处车声、风声、街道喧闹

如果后续能加入音效，整个游戏会更像一个真正的动作游戏而不只是演示画面。

---

## 5. 关卡内容扩展建议

### 5.1 关卡主题

可以把当前单一街道改为多个主题场景：

- 夜间街区：初始关卡，普通敌人较多
- 旧巷道：中段，视野更窄、敌人更密
- Boss 区域：终局，灯光和环境压迫感更强

### 5.2 关卡目标

除了“推进到终点并打败 Boss”，可以加入更多目标：

- 保护某个目标点
- 在限定时间内清理敌人
- 通过某个障碍物或机关
- 在 Boss 战前完成小段过关任务

## 6. 适合按当前工程结构推进的任务

### ViewModel 层

适合增加的内容：

- 更细致的攻击判定逻辑
- Boss 阶段切换逻辑
- 连击、受击、硬直、击退效果
- 资源道具系统
- 难度曲线与敌人生成规则

### View 层

适合增加的内容：

- 动作动画与特效渲染
- Boss 出场动画和屏幕震动
- UI 动画和按钮反馈
- 更细致的背景与粒子视觉

### Common 层

适合增加的内容：

- 新的状态类型，如受击、格挡、冲刺、Boss 阶段
- 更结构化的敌人配置数据
- 资源道具与事件类型定义

## 7. 推荐优先级

### P0（最值得先做）

- Boss 遭遇动画和锁屏效果
- 更清晰的攻击反馈（命中、受击、击退、硬直）
- 进度条与 Boss 血条UI优化
- 简单粒子特效与屏幕震动

### P1（下一阶段建议）

- 新敌人类型与敌人波次设计
- 血包/精力补给道具
- 标题页和结算页美化
- 音效接入

### P2（后续扩展）

- 多关卡场景
- 更多 Boss 阶段和技能
- 设定、剧情以及成就系统

---

## 8. 一份较实际的迭代节奏建议

### 第一周：战斗反馈与 Boss 体验

- Boss 出场动画
- 攻击命中/受击反馈
- 简单粒子特效

### 第二周：关卡与敌人设计

- 新敌人类型
- 波次设计
- 补给道具

### 第三周：视觉与 UI 提升

- 背景美化
- HUD 美化
- 结算界面与标题页设计

### 第四周：音效与内容收束

- 音效接入
- 关卡节奏调整
- 平衡与打磨

---

## 9. 结论

下一轮迭代最值得做的不是“再加几个按钮”，而是把当前已有规则变得更完整：

- 让 Boss 战更有冲击
- 让战斗更有反馈
- 让关卡更有节奏
- 让画面和 UI 更有“游戏感”

如果按这个方向推进，项目就会从“可演示原型”更自然地走向“完成度较高的 2D 街机动作游戏雏形”。
