# 环境与构建

本项目使用 CMake + vcpkg 管理构建和第三方依赖。当前 `vcpkg.json` 已声明 FLTK 依赖，后续如果增加音效库、图片库等，也统一加到 `vcpkg.json`。

首次配置环境：

```bash
git clone https://github.com/microsoft/vcpkg.git ~/vcpkg
~/vcpkg/bootstrap-vcpkg.sh
export VCPKG_ROOT=~/vcpkg
```

Windows 下对应设置 `VCPKG_ROOT` 到自己的 vcpkg 目录，并运行 `bootstrap-vcpkg.bat`。

配置和编译：

```bash
cmake --preset vcpkg
cmake --build --preset vcpkg
```

如果 IDE 支持 CMake Presets，直接选择 `vcpkg` preset 即可。

# 第一轮迭代：基础框架与界面设计
第一轮首先完成Common 层的编写，随后并行开发 view 层和 ViewModel 层，最后在 app 层完成跑动与攻击动画的显示；
- Common 层：存放数据结构。这里在 common 层中放置玩家、小怪、boss 以及背景地图四个类的声明以及定义。在 common 层完成编写后 view 层和 viewmodel 层的编写将根据common 层中定义的数据结构分别独立进行。
- View 层：view 层中隔离地实现了底层的图片管理（Animation 类、ResourceManager 类）与动画播放（draw 方法），同时也实现了键盘事件的响应。
- ViewModel 层：加载和改变需要呈现的各种数据，提供 common 对象集合，提供可用来显示的数据类，提供了一系列 get,set 函数暴露可用来绑定的数据属性。
- App 层：创建 common 层、view 层、ViewModel 层的对象，并实现它们之间的相互绑定。

# 游戏规则与玩法介绍
1. 核心胜利目标
扮演一名格斗家，从关卡的最左端一路打到最右端。途中会遭遇多波敌人的拦截，玩家需要用拳脚击败所有杂兵，并在关底击败最终的 Boss，即可获得整场游戏的胜利。
2. 玩家的控制与操作
- 移动： 玩家可以控制角色在一条有纵深的街道上进行上下左右八个方向的移动。虽然屏幕是平面的，但角色可以通过向上或向下走，来躲避敌人的正面攻击。
- 基础攻击（轻功/重击）： 玩家拥有基础的连招规则。比如连续按下轻拳键，角色会自动打出一套“左勾拳+右勾拳+一记重摔”的组合拳。
- 跳跃与空中攻击： 玩家可以跳起躲避地面的横向攻击，或者在跳起时按下攻击键，使出“飞踢”来切入战场。
3. 状态与资源规则（怎么算输？）
- 血条（生命值）： 屏幕上方有玩家的血条。玩家每被敌人打中一次，血条就会扣减。如果血条彻底清空，游戏立刻结束（Game Over）。
- 精力条（动作限制）： 玩家在连续挥拳、跑动或使用强力技能时，会消耗精力条。精力条耗尽时，玩家会进入短暂的疲劳或者无法连续出拳，需要站在原地等待它自动恢复。这要求玩家不能盲目“脸滚键盘”瞎按，得注意节奏。
4. 关卡推进规则：硬性推图锁
为了防止玩家不打怪直接一路飙车跑到终点，游戏制定了严格的“推图解锁”规则：
- 遭遇战锁定： 当玩家走到街道的某个特定位置时，屏幕会突然“卡住”无法再往右走。同时，场景的左侧和右侧会刷出几个小怪包围玩家。
- 清屏解锁： 玩家必须把当前屏幕里的所有小怪全部打倒。只有当场上一个敌人都不剩时，屏幕才会解锁，并亮起一个闪烁的“GO ▶”图标，允许玩家继续向右走。
5. 敌人的行为规则
为了让闯关有挑战性，场上的敌人有不同的行为模式：
- 普通小怪（包抄型）： 数量较多，血量较薄。它们会主动向玩家靠近。好玩的是，它们不会傻傻地重叠在一起，而是会尝试“上下包抄”，比如一个从上面绕到你身后，一个在前面吸引火力。
- 首领（Boss）： 在关卡的最后出现。血量极厚，攻击力极高，且拥有更强的进攻欲望。它会死死盯住玩家的位置进行持续的追击，是玩家通关的最大障碍。
6. 重开循环
如果玩家在半路不幸被小怪或者 Boss 打死，游戏会跳出结算界面。玩家可以选择点击“重新开始”，此时游戏的所有状态（血量、关卡进度、怪物数量）会全部重置，玩家回到最左端的起点重新挑战.
